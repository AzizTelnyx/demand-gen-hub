# LiveKit / Voice AI Infrastructure Campaign

## Objective
Acquire net-new customers building real-time voice agents who are already using LiveKit-style frameworks.

## Why Now
- AI components (STT, TTS, LLMs) now dominate cost and performance in voice stacks
- Developers design around agents, not telephony primitives
- Customers face rising AI bills, fragmented vendors, latency issues
- Telnyx can offer lower latency, consolidated models, better economics

## What Telnyx Offers
Lower-cost, lower-latency execution layer for real-time voice agents with access to best-in-class STT, TTS, and inference through single integration.

**Enabled by:**
- Telephony, media, and speech services on same network
- Fewer network hops between audio ingestion, AI processing, output
- Aggregated access to multiple AI models

## ICP (Ideal Customer Profile)

**Primary:**
- Teams building real-time voice agents
- Using LiveKit, livekit/agents, Pipecat, or similar frameworks
- Experiencing: high STT/TTS costs, latency issues, vendor complexity

**Typical profiles:**
- AI-native startups
- Contact-center AI platforms
- Engineering-led teams with custom voice stacks

## Core Value Proposition

1. **Lower cost at scale** — Aggregated pricing for STT, TTS, inference through single provider
2. **Lower end-to-end latency** — Same network, dedicated infrastructure, fewer hops
3. **Simple architecture** — Replace fragmented stacks with single execution layer
4. **Future flexibility** — Switch between STT/TTS engines without re-architecting
5. **Open ecosystem** — Public Slack, active forums, 24/7 support, deep docs
6. **Agent-ready foundation** — Built for agent-first model (evolving)

## Campaign Themes
- Unified voice AI infrastructure for optimal performance
- Cost efficiency at scale
- Latency and performance gains
- Simplifying AI voice stacks
- Agents as the natural abstraction

## Sales Alignment

**Core message:**
> Telnyx makes real-time voice agents cheaper, faster, and more reliable by running telephony, media, and AI services together in the call path.

**Sales should:**
- Lead with cost and latency improvements
- Position Telnyx as execution layer in real-time call path
- Emphasize improving existing agents without rebuild
- Frame Call Control as transport layer, not something to script

**Discovery questions:**
- "What are you using today for STT and TTS?"
- "How many vendors sit in your real-time call path?"
- "Where do you see cost or latency issues today?"

## Internal Alignment
Focused on winning net-new customers running real-time voice agents by providing better infrastructure layer for what they've already built.
